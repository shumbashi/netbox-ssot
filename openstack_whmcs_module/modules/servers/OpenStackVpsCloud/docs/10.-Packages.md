### Czym są paczki?
Są to już przygotowane, gotowe elementy, które można użyć w module. 

### Jaki ich używać?
Dodaj do pliku `composer.json` podobny kod
```json
  "extra": {
    "packages": [
      "DevTools"
    ]
  }
```
Gdzie w `packages` wymienisz listę paczek które chcesz zainstalować. Następnie uruchom `composer update && composer run-script fw-install`.
`Uwaga!` Windows lubi robić problemy, więc zanim uruchomisz `composer update` usuń katalog `vendor`. 

### Struktura paczki
Wszystkie paczki są umiesczone w katalogu `packages`. Głównym plikiem inicjalizujacym paczkę jest plik `install.php`, który odpowiada za inicjalizacje elementów paczki oraz wpięcie jej różny miejsca we frameworku. Pozostałe pliki powinny zachować strukturę znaną z katalogu `app`. 

### Konfiguracja paczki
```php
use ModulesGarden\OpenStackVpsCloud\Core\DependencyInjection\Container;
use ModulesGarden\OpenStackVpsCloud\Packages\Logs\Http\Admin\LogsPage;
use ModulesGarden\OpenStackVpsCloud\Packages\Logs\Logs;

return [
    'enable'      => true,
    'menu'        => [
        'admin'  => [
            'logsPage' => [
                'icon' => 'lu-nav__link-icon lu-zmdi lu-zmdi-home'
            ]
        ],
        'client' => [

        ]
    ],
    'controllers' => [
        'admin'  => [
            LogsPage::class
        ],
        'client' => [

        ]
    ],
    'bootstrap'   => function () {
        Container::getInstance()->alias(Logs::class, "logger");

        \ModulesGarden\OpenStackVpsCloud\Core\listen(\ModulesGarden\OpenStackVpsCloud\App\Events\Events\ModuleActivated::class, \ModulesGarden\OpenStackVpsCloud\Packages\Logs\Listeners\ModuleActivated::class);
    }
];
```
W pliku wyróżniamy takie właściwości jak:
- enable - true/false - definiuje czy paczka jest aktywna
- menu - definiuje menu jakie może zostać dodane do menu aplikacji
- controller - informuje framework o odstępnych kontrolerach
- boostrap - jest to funkcja którą uruchamia framework, za jej pomocą może się wpiąc w inne miejsca we frameworku 

### Wpięcie we frameworka
#### Rejestracja własnej klasy która będzie dostępna w DI 
```php
Container::getInstance()->alias(Logs::class, "logger");
```
#### Rejestracje zdarzeń
Pozwala na wykonanie pewnych czynności na zdarzenia które dzieją się w systemie. Dostępne zdarzenia znajduja się w przestrzeni `\ModulesGarden\OpenStackVpsCloud\App\Events\Events\`. Przykładowym zdarzeniem która pozwala na utworzenie własnej tabeli jest:
```php
        \ModulesGarden\OpenStackVpsCloud\Core\listen(\ModulesGarden\OpenStackVpsCloud\App\Events\Events\ModuleActivated::class, \ModulesGarden\OpenStackVpsCloud\Packages\Logs\Listeners\ModuleActivated::class);
```
#### Hooki WHMCS
Paczka może sie wpiąc w hooki WHMCS. W tym celu należy uruchomić kod 
```php
HookManager::create(__DIR__, true);
```
oraz stworzyć katalog `Hooks` który będzie zawierał hooki zdefiniowane w taki sam sposób jak ma to miejsce w `app`. 

#### Langi
Langi są automatycznie ładowane z katalogu `langs`. Należy zachować te same zasady co w przypadku `app`. Paczki nie wspierają nadpisywanie langów poprzez katalog `overrides`.
